1) Версия этапа: v0.5 (UI Usability Pack)

2) Ключевые изменения:
- Новый helper app/tools/ui_utils.py: copy/paste, бинды Ctrl+C/V/A/X для Text/Entry, открытие путей в проводнике, установка статуса (логируется в launcher.log).
- Launcher GUI: во всех текстовых полях добавлены кнопки Копировать/Вставить/Очистить/Сохранить/Открыть, многострочные поля переведены на Text+Scrollbar, глобальный статус-бар показывает последнее действие.
- Вкладка "Тест AI": prompt/response теперь отдельные текстовые блоки с копированием и сохранением в файл.
- Вкладка "Симулятор": история со скроллом, копируемая, с кнопкой "Копировать весь лог симуляции" и поддержкой сохранения/загрузки.
- Вкладка "Настройки": поля BOT_TOKEN/OPENAI_API_KEY со скрытием/показом, копированием/вставкой, открытием .env и bot_overrides.json из GUI.
- Вкладка "Запуск": лог бота со скроллом, кнопки копирования/вставки/очистки/сохранения и быстрый переход к файлу логов.

3) Как запускать на Windows 10:
- Двойной клик по START_ALL.bat (создаёт .venv, ставит deps, открывает .env при отсутствии, запускает GUI). Лог: logs/setup.log.
- Запустить GUI вручную: START_LAUNCHER.bat. CLI: .venv\Scripts\python.exe launch.py --run-bot

4) Где смотреть логи и конфиг:
- logs/setup.log — батники/установка.
- logs/launcher.log — GUI + все действия копирования/открытия/сохранения.
- logs/app.log — бот.
- .env и bot_overrides.json можно открыть из GUI кнопками.

5) Обязательные проверки (Windows):
- Ctrl+C/Ctrl+V работают в Text и Entry (бинды в ui_utils.bind_clipboard_shortcuts).
- Все кнопки "Копировать" копируют содержимое соответствующего поля.
- Кнопка "Открыть логи" в "Диагностика" открывает папку logs.
- Кнопка "Сохранить ответ"/"Сохранить" в AI/симуляторе/логах сохраняет в .txt (file dialog), затем файл можно открыть и прочитать содержимое.

6) Примечания/риски:
- Ошибки действий (копирование/открытие файлов и т.п.) показываются через messagebox и логируются в logs/launcher.log.
- OpenAI/Telegram проверки требуют валидных токенов и сети; при ошибке статус-бар и лог покажут причину.
- Python 3.14 по-прежнему не поддерживается; использовать py -3.12.
